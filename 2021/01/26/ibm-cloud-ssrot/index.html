免费容器 IBM Cloud 搭建 SSRot 代理

发表于 2021-01-26 更新于 2021-08-17 分类于 爬墙 阅读次数： 本文字数： 1.4k 阅读时长 ≈ 2 分钟

说明：

  1. IBM Cloud 为我们提供了免费的容器服务，我们不应该滥用它，所以不宜做为长期翻墙使用。
  2. IBM Cloud 免费版有所限制，几天无人访问就会休眠。（具体是几天已经忘记了，不过下面讲述的方法会每天定时重启实例，所以无需担心）

正文：(IBM Cloud 注册过程就不赘述了，注册不成功，就换邮箱、IP)

  • 点击右上角 创建资源 ，点击 Cloud Foundry，点击 创建 。
  • 达拉斯地区的 域 要选择 us-south.cf.appdomain.cloud，伦敦地区的 域 则要选择 eu-gb.cf.cloud.ibm.com。其它的就随意填写、选择。（记住你填写的应用程序名称）最后点击创建。
  • 创建完后会自动跳转到控制台，点击右上角的 IBM Cloud Shell。
  • 如果你是伦敦地区的需要输入以下代码强制切换命令行所在的地区。
  • 输入以下代码，下载 SSRot 的脚本文件。
  • 输入以下代码，运行脚本来安装 SSRot。
  • 之后会提示是否安装 cloud foundry cli，选择 y。提示输入你的应用程序名称，提示输入应用的内存大小。（免费版的最大内存为256MB）
  • 之后会提示输入密码和路径，可以直接回车。
  • 安装完后，会把配置文件直接输出到命令行，记录下密码和路径。
  • 点击下方链接进入 ssrot-ibm 项目地址。 https://github.com/zyp997/ssrot-ibm
  • 点击右上角 Fork ，会跳转到你的项目地址。（需要 GitHub 账号）
  • 点击 Settings ，在左边栏目找到并点击 Secrets ，再点击右上角的 New repository secret，分别创建如下表的参数。
Secrets变量说明
MAILIBM Cloud 邮箱地址
PWDIBM Cloud 邮箱密码
CFNAMEIBM Cloud 应用程序名
RGNus-south(达拉斯)、eu-gb(伦敦)
  • 点击 Actions ，点击 I understand my workflows, go and ahead and enable them ，在左边栏目找到并点击 IBM Cloud Auto Restart ，点击 Enable workflow ，再点击 Run workflow 选项，最后点击 Run workflow 。
  • 在左边栏目找到并点击 All workflows ，如出现下图结果则表示配置成功。(如果什么都没显示，则在 README.md 文件随便打个字符即可；如果执行失败，则仔细检查 Secrets 变量 是否填写正确，之后再重新执行 Actions)
  • 下载支持 SSRot 协议的客户端，配置信息如下：

地址：IBM Cloud 应用程序名.us-south.cf.appdomain.cloud //伦敦地区的为 IBM Cloud 应用程序名.eu-gb.cf.appdomain.cloud
端口：443
密码：上面记录的密码
加密方式：aes-128-ctr
传输协议：auth_aes128_md5
混淆插件：tls1.2_ticket_auth
路径：/上面记录的路径/
底层传输安全：tls
