表单

常用表单元素属性（input 标签）

  • type(必填)：指定元素的类型。text(文本框)、password(密码框)、checkbox(多选框)、radio(单选框)、submit(提交按钮)、reset(重置按钮)、file(文件)、image(图片)、button(按钮)、email(邮箱)、url(网址)、number(数字)、range(滑块) 和 search(搜索框)。
  • name：指定表单元素的名称。（注意：虽然有一小部分表单元素不设置 name 属性也没事，但是几乎全部都是必须设置的，所以建议所有表单元素都设置 name 属性！）
  • value：为表单元素设定值。type 为 radio 或 checkbox 时，必须设置 value 属性。对于不同的输入类型，value 属性的用法也不同：
    • type 为 button、reset、submit - 定义按钮上的显示的文本
    • type 为 text、password、hidden - 定义输入字段的初始值
    • type 为 checkbox、radio、image - 定义与输入相关联的值
    • type 为 file 时不适用
  • size：指定表单元素的初始宽度。当 type 为 text 或 password 时，表单元素的大小以字符为单位。对于其他类型，宽度以像素为单位。
  • maxlength：type 为 text 或 password 时，输入的最大字符数。
  • checked：type 为 radio 或 checkbox 时，指定按钮是否是被选中。
  • hidden：隐藏元素（只是隐藏了但是还存在）
  • disabled：禁用（表现为无法输入、选择、点击和调节元素）
  • readonly：只读元素（表示输入框只读，不能进行输入和修改。其它的元素也能使用 readonly 属性，表现为类似 diasbled 属性的效果）
  • placeholder：输入框提示信息。
  • required：必填项（表示元素一定要输入或选择，否则就无法提交表单）
  • pattern：正则表达式

下拉框和文本域（非 input 标签）

  • 下拉框：select - option
  • 文本域：textarea - cols - rows

表单提交

  • action：表单提交的位置，可以是网站，也可以是一个请求处理地址。
  • method：post、get 提交方式。get：在网址中会看到提交的信息，不安全却高效。post：比较安全，可传输大文件。
